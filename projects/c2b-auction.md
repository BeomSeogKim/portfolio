# C2B 경매 서비스 개발

> 2025.06 · 백엔드 서버 설계 및 전체 API 개발 (기여도 100%)

`Java` `Spring Boot` `JPA`

**목표 기간 내 MVP 무장애 출시, 테스트 커버리지 100% 달성**

<br/>

## Background

- 위탁 판매 가격에 불만족하거나 판매가 장기간 이뤄지지 않는 고객에게 새로운 판매 창구를 제공하여 리텐션을 높이기 위해 C2B 경매 시스템 도입
- 확정되지 않은 기획 상태에서 3주 내 경매 시스템 MVP 출시 필요
- 기획 변경이 빈번하여 핵심 비즈니스 규칙(재경매 가능 조건, 위탁 상품 경매 가능 조건, 회차별 입찰 내역 저장 등)이 지속적으로 변동

<br/>

## Approach

### 아키텍처 설계

- 빈번한 기획 변경에도 핵심 로직을 명확히 유지하기 위해 헥사고날 아키텍처 도입
- 외부 의존성과 핵심 도메인 로직을 분리하여, 기획 변경 시 영향 범위를 최소화

### 도메인 모델 문서 기반 개발

- 초기 단계에서 도메인 모델 문서를 작성해 기획/실무자와 합의 후 개발
- 요구사항 변경 시 문서 업데이트 → 합의 → 구현 → 테스트코드 작성의 일관된 프로세스로 유지보수 효율성 향상

### 사용자와 도메인 간 관심사 분리

- 경매 서비스에 세 가지 유형의 사용자 존재 (판매자, 구매자, 관리자)
- 도메인별로 각 사용자에 대한 비즈니스 요구사항이 상이
- 도메인 서비스를 사용자별 서비스 클래스에서 조립하여 사용

### 후속 작업 관심사 분리

- 경매 진행 과정에 따라 발생하는 후속 작업을 Spring Event 기반 도메인 이벤트로 분리 처리
- 도메인의 순수성 및 응답성 확보

<br/>

## Results

- 3주 내 MVP 무장애 출시
- 핵심 도메인 로직 테스트 커버리지 100%
- 출시 이후 누적 경매 등록 약 4,000건
- 출시 이후 기획 변경 요청에 대해 역할 기반 분리 구조로 빠르게 대응
