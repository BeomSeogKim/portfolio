# 쿠폰 서비스 성능 개선

> 2024.10 · 기여도 100%

`Java` `Spring Boot` `JPA` `MySQL`

**API 응답 속도 최대 97% 단축 및 서비스 안정성 확보**

<br/>

## Background

- 네이버 쇼핑 등 외부 채널 입점 시 가격 경쟁력 확보를 위해 상품별 쿠폰 부착 정책 도입
- 이로 인해 활성 쿠폰 수가 90배 증가 (100개 → 9,000개)
- 정책 변경 후 Grafana 모니터링 중 주요 쿠폰 조회 API의 응답 시간이 최대 12초까지 지연되는 것을 발견
- 상품 상세 화면의 쿠폰 후보 조회, 결제 화면의 사용 가능 쿠폰 조회 등 구매 핵심 경로에 위치한 API가 병목

<br/>

## Approach

### 병목 분석

- Grafana에서 Latency가 걸리는 API 식별
- 각 API에 대해 StopWatch를 사용하여 병목 지점 식별

### N+1 문제 해결

- Fetch Join을 적용, 연관관계 엔티티 조회 시 발생하던 다중 쿼리를 단일 쿼리로 최적화

### DB 조회 최적화

- 애플리케이션 레벨에서 각 쿠폰의 유효성을 개별 쿼리로 검증하던 로직(N+1)을, 모든 조건을 포함한 단일 네이티브 쿼리로 개선

<br/>

## Results

| API | Before | After | 개선율 |
|:----|:-------|:------|:------|
| 전체 쿠폰 조회 | 12s | 0.3s | 40배 향상 |
| 다운로드 가능한 쿠폰 조회 | 6.5s | 0.3s | 22배 향상 |
| 사용 가능한 쿠폰 조회 | 5s | 1s | 5배 향상 |

- 결제 프로세스 내 병목 해소로 구매 전환율 저하 방지
