# 기획전 서비스 성능 고도화

> 2025.09 · 병목 분석 및 성능 개선 단독 수행

`Kotlin` `Spring WebFlux` `Coroutine` `Redis`

**대규모 기획전(3,000개 상품) 응답 속도 93% 개선 (30s → 2s)**

<br/>

## Background

- 사내 Server Alert Slack에서 기획전 조회 API Latency 경고 발생
- 대규모 상품이 포함된 기획전 조회 시 응답 시간 최대 30초까지 지연

<br/>

## Approach

### 비동기 안티 패턴 식별 및 병목 분석

- 프로파일링을 통해 병목 지점 식별: 청크 단위 외부 API 호출을 루프 내 `awaitSingle()`로 처리하는 직렬 대기 패턴 발견
- 비동기 프레임워크(WebFlux)를 사용하면서도 실제 로직은 동기적으로 수행되어 네트워크 I/O Latency 누적

### 구조화된 동시성 기반의 이중 병렬화 구현

- **모듈 레벨:** 서로 의존성 없는 기획전 내 모듈(캐러셀, 초저가 등)을 `coroutineScope` 내 `async`로 분리하여 병렬 처리
- **데이터 레벨:** 상품 상세 조회를 Scatter-Gather 패턴으로 재설계, 상품팀과 합의한 chunk 사이즈(25개)로 병렬 호출
- **부분 실패 허용:** 외부 API 에러 시 해당 상품만 제외하고 나머지 정상 응답

### Cache Refresh Ahead 전략 도입

- 단순 TTL 만료 시 발생하는 캐시 미스와 Latency Spike 방지
- TTL 1분 미만 시점에 백그라운드에서 비동기로 데이터 갱신, 사용자에게는 항상 빠른 응답 보장
