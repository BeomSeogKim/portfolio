# 대용량 유저 행동 데이터 처리 시스템 구축 및 최적화

> 2024.10 ~ 2024.12 · 기여도 100%

`Java` `Kafka` `Spring Boot`

**데이터 처리 최대 지연 시간 20시간 → 10초 이내 (99% 이상 단축)**

<br/>

## Background

- WebLog Service가 수집하는 유저 행동 로그를 Kafka 이벤트로 받아, BytePlus 추천 시스템 연동에 필요한 양식으로 변환·전송하는 파이프라인
- 일 평균 300만 건의 행동 데이터 처리가 지연되어 최대 20시간까지 적체, 추천 품질에 직접적 영향
- 원인 분석 결과, 메시지마다 반복적으로 수행되는 상품 API 호출 및 내부 암호화 로직의 I/O 오버헤드가 누적되어 병목 발생

<br/>

## Approach

### 처리 단위 변경

- `Kafka Batch Listener`를 활용, 메시지 처리 단위를 개별에서 배치로 전환하여 I/O 호출 횟수를 최소화

### Bulk API 호출

- N개의 메시지에 대해 발행하던 N번의 개별 상품 API 호출을, Bulk API 호출로 통합하여 네트워크 오버헤드 감소
