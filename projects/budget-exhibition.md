# 기획전 위젯 개발

> 2024.06 ~ 2026.01 · 전시팀 단독 설계 및 개발 (기여도 100%)

`Java` `Spring Boot` `MySQL`

**기획전 페이지 구매 전환율 40% 성장, 운영 리소스 월 20시간 이상 수작업 100% 자동화**

<br/>

## Background

- 트렌비 상품은 카탈로그 단위로 묶여 있으며, 동일 카탈로그 내에서 가격이 가장 낮은 초저가 상품이 존재
- 이 초저가 상품들을 기획전으로 자동 노출하여 판매율을 높이자는 전략적 목표
- 기존에는 MD가 수작업으로 상품 카탈로그 번호를 입력하여 저가 상품들을 진열 → 월 20시간 이상 소요
- 상품팀에서 초저가 상품 목록을 제공하는 API를 개발, 전시팀(본인)이 기획전 위젯 및 자동화 프로세스를 설계·개발

<br/>

## Approach

### 초저가 기획전 위젯 자동화 (2024.06~07)

- 초저가 상품 전용 위젯을 별도 생성하여 기획전 페이지에 진열
- CronWorkflow를 활용하여 12시간 주기로 상품팀 API에서 초저가 뱃지 상품을 조회, 위젯에 자동 업데이트

### 대량 데이터 문제 해결

- 자동화 이후 초저가 대상 상품 수가 급증하며 MySQL에서 `Packet for query is too large` 오류 발생
- 상품팀 API가 제공하는 목록 기준 상위 1,000개 상품만 전시하도록 조회 범위를 제한하여 해결

### 기획전 캐러셀 위젯 추가 개발 (2026.01)

- 상품 서비스 API를 호출하여 데이터를 받아오는 구조로 다양한 기획전 위젯 개발
- 비디오 캐러셀: 비디오 정보가 포함된 상품을 기획전에 자동 노출 (상품팀 협업)
- 자동화 캐러셀: 조건 기반 상품 자동 진열
- 타임딜 캐러셀: 시간 제한 프로모션 상품 진열
- 탭형 캐러셀: 카테고리별 탭 구조로 상품 진열

<br/>

## Results

- 기획전 페이지 구매 전환율 40% 성장
- 월 20시간 이상 소요되던 MD 수작업을 100% 자동화
- 다양한 형태의 기획전 위젯을 통해 상품 노출 방식 다각화
